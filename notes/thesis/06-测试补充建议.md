## 一、建议优先级最高

### 1. 聊天主流程端到端测试

建议新增目录：

```text
tests/preview/
```

建议文件：

```text
tests/preview/chat-build-smoke.spec.ts
```

建议覆盖点：

- 打开首页后聊天界面可正常渲染
- 输入一个简单提示词后消息能正常发送
- 返回消息中出现结构化产物或普通文本响应
- 工作台出现文件或预览区域可见

原因：

这是整个平台最核心的闭环链路，也是论文里最有说服力的测试。

### 2. 消息解析到动作执行链测试

建议文件：

```text
app/lib/runtime/action-runner.spec.ts
app/lib/stores/workbench.spec.ts
```

建议覆盖点：

- 接收到 file action 后是否正确写入文件
- 接收到 shell/start/build action 后状态是否从 `pending` 进入 `running` 再进入 `complete`
- 失败时是否正确产生 `failed` 状态和错误信息

原因：

当前最关键的自动化链路之一，但仓库里还没有对应测试。

### 3. 会话与快照恢复测试

建议文件：

```text
app/lib/persistence/useChatHistory.spec.ts
app/lib/persistence/db.spec.ts
```

建议覆盖点：

- 新会话消息是否能正确保存到 IndexedDB
- 根据 `chatId` 是否能正确恢复消息
- 快照恢复时是否能重建文件状态

原因：

这是系统连续使用能力的重要支撑，也是论文中“可恢复性测试”的关键依据。
